GXO Fabric Modernisation Case Study

Scoped contribution to dataset modernisation in Microsoft Fabric, focused on reliability, semantic clarity, and practical controls.

Context

Migration and modernisation of legacy reporting datasets in a live operational environment.

Role

Senior MI Analyst, contributing BI and data engineering components.

Tools / Stack

Microsoft Fabric, Lakehouse patterns, PySpark notebooks, SQL, Power BI semantic model, DAX Studio.

Ownership / Scope

Within my scope: ERD maintenance, data-flow documentation, reconciliation checks, DAX performance review, and runbook support.

Constraints

No disruption to operational reporting, evolving KPI definitions, and governance requirements on documentation and controls.

Problem

Reporting logic had developed across legacy assets over time, creating inconsistency in metric definitions and duplicated transformation logic. The workstream required modernisation toward maintainable Fabric patterns while preserving day-to-day reporting continuity.

Constraints

  • Operational stakeholders required uninterrupted reporting during transition phases.
  • KPI expectations changed as teams aligned on definitions across functions.
  • Documentation needed to be audit-friendly and suitable for handover.
  • Decisions on refresh and retention were made with wider team input; I supported these decisions within my remit.

Approach

  • Contributed to migration activities by helping move legacy logic toward layered Lakehouse-aligned patterns.
  • Delivered components such as ERD maintenance and data-flow diagrams to keep entity, key, and metric logic visible.
  • Partnered with stakeholders to align KPI interpretation and keep documentation current as logic evolved.
  • Supported ingestion and transformation work in PySpark notebooks and SQL for stable reporting inputs.

Validation / controls

  • Implemented reconciliation checks across source, transformed, and semantic outputs for key metrics.
  • Supported control routines for refresh exceptions and threshold variances.
  • Contributed to runbook updates so checks and escalation paths were usable by the wider team.

Semantic model & DAX performance pass

  • Reviewed semantic relationships and measure structure to keep logic maintainable.
  • Used DAX Studio to review query behaviour and support targeted measure tuning.
  • Shared optimisation notes with trade-offs so improvements were clear and repeatable.
Example artefact: DAX Studio performance notes (placeholder).

Outcomes

  • Improved reliability through clearer data flow, documented definitions, and practical reconciliation controls.
  • Supported more consistent KPI interpretation by keeping ERDs, data flow, and metric documentation aligned.
  • Contributed to delivery hygiene through runbook and documentation updates that reduce single points of failure.

Next steps

  • Continue incremental migration of legacy logic into standardised Fabric patterns.
  • Expand control coverage for additional high-impact metrics.
  • Revisit semantic model hotspots periodically using DAX Studio as usage patterns evolve.
