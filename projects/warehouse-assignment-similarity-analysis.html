Warehouse Assignment Similarity Analysis

Supporting Python EDA project analysing SKU and location similarity across multi-container warehouse assignments to identify potential batching inefficiencies and operational improvement opportunities.

PythonpandasmatplotlibseabornEDAJaccard SimilarityWarehouse AnalyticsOperational Efficiency

Overview

Analysed warehouse multi-container picking assignments to assess whether containers grouped into the same assignment were genuinely similar by SKU and location. The analysis used Python EDA to quantify clustering quality and identify possible inefficiencies in assignment batching.

Business question

Were containers grouped into the same picking assignment similar enough to support efficient picking, or were assignments combining poorly matched containers that could increase travel distance and completion time?

Data and scope

  • Picking activity records from a warehouse operation.
  • Assignment identifier, container identifier, SKU, consignment grouping, pick location and timestamp fields.
  • Initial dataset was approximately 3.26M records.
  • Analysis focused on valid assignments containing exactly 3 containers.
  • Final comparison covered approximately 37,659 assignments.

The underlying notebook/export is not publicly linked from this case study because it contains operational field names and low-level working detail. This page summarises the anonymised business problem, method and findings only.

Methodology

  • Used pandas for data loading, filtering, grouping and feature engineering.
  • Derived aisle/location groupings from pick locations.
  • Compared containers within each assignment using Jaccard similarity.
  • Calculated SKU similarity, location similarity and aisle similarity.
  • Used pairwise comparison across the 3 containers in each assignment.
  • Aggregated average and minimum similarity scores.
  • Used minimum similarity as an inefficiency signal because one poorly matched container can weaken the whole assignment.
  • Used matplotlib/seaborn for exploratory visualisation where relevant.

Key findings

  • Average SKU similarity was approximately 12.5%, suggesting weak product clustering across assignments.
  • Around 76% of assignments contained at least one poorly matched container.
  • Higher similarity was associated with faster completion times and reduced variability.
  • Location similarity was also important, indicating that physical proximity matters as much as SKU overlap.
  • Consignment grouping alone did not appear to guarantee efficient similarity or performance.

Tools and techniques

  • Python for exploratory analysis and repeatable calculation logic.
  • pandas for filtering, grouping, joining and feature engineering.
  • Jaccard similarity for SKU, location and aisle-set comparison.
  • matplotlib and seaborn for exploratory visualisation and distribution checks.

Operational value

  • Created a clearer way to evaluate assignment batching quality.
  • Helped identify where current grouping logic may create avoidable travel or fragmented work.
  • Provided a basis for discussing SKU/location-aware clustering improvements.
  • Demonstrated how Python analysis can move beyond reporting into operational diagnostics.

Limitations

  • Location similarity is a proxy for travel efficiency, not a full route optimisation model.
  • Analysis shows association, not necessarily full causation.
  • Completion time can be affected by other factors such as picker behaviour, congestion, stock availability or exception handling.
  • Next step would be validating proposed batching changes with operational teams and/or controlled pilots.

What this demonstrates

  • Python EDA in a workplace setting.
  • pandas-based data preparation and feature engineering.
  • Similarity metrics/Jaccard similarity.
  • Analytical framing of an operational efficiency problem.
  • Translating technical outputs into business recommendations.
  • Awareness of limitations and operational context.
