SKU Similarity Efficiency in Multi-Container Assignments¶

Executive Summary¶

This analysis evaluates the effectiveness of assignment batching in warehouse operations by measuring similarity between containers within each assignment.

Key findings include:

  • Low overall similarity: Average SKU similarity across assignments is approximately 12.5%, indicating weak clustering.
  • High inefficiency prevalence: ~76% of assignments contain at least one poorly matched container.
  • Performance impact: Higher similarity is associated with faster completion times and reduced variability.
  • Location matters as much as SKU: Physical proximity (location similarity) is as influential as SKU similarity in determining efficiency.
  • Consignment grouping is ineffective: Assignments grouped by consignment do not show improved similarity or performance.

These findings suggest that current assignment logic can be significantly improved by incorporating both SKU and location-based clustering.

Background¶

In warehouse operations, each LLOP (Low-Level Order Picker) assignment consists of three containers (cages) grouped under a single assignment identifier (LIST_ID).

Although these containers are picked together, they may:

  • Belong to different consignments (customers)
  • Be delivered to different loading lanes

The intended picking strategy is to cluster similar SKUs within the same assignment to:

  • Reduce travel distance
  • Improve picking efficiency
  • Enable faster completion

However, there is currently limited visibility into how effectively this clustering is achieved in practice.

Problem Statement¶

The current assignment process does not provide clear insight into how similar containers are within the same assignment.

This creates several potential inefficiencies:

  • Containers with low SKU overlap may require pickers to travel across multiple locations
  • Poor clustering can lead to inefficient pick paths
  • Assignments may take longer to complete due to fragmented work
  • Some containers may remain incomplete longer, delaying overall completion

This analysis aims to quantify and evaluate these inefficiencies.

Key Analytical Questions¶

The analysis is designed to answer the following:

  1. What is the percentage similarity of SKUs between containers within each assignment?

  2. How does similarity vary depending on consignment structure?

    • All 3 containers from different consignments
    • 2 containers from one consignment and 1 from another

  3. Are higher similarity scores linked to:

    • Faster assignment completion?
    • Reduced picker travel (approximated through location similarity)?

  4. What proportion of assignments show low similarity and may indicate inefficiency?

Data Preparation & Business Logic¶

To ensure consistent comparison:

  • Only assignments with exactly 3 containers are retained
  • SKU sets are compared across these containers
  • Location information is used to approximate travel efficiency

Additional features:

  • Aisle: derived from location
  • Consignment group: derived for classification purposes

Methodology¶

Similarity is measured using Jaccard similarity:

J(A, B) = |A ∩ B| / |A ∪ B|

Applied to:

  • SKU sets (product overlap)
  • Location sets (spatial proximity)
  • Aisle sets (coarse spatial grouping)

For each assignment:

  • Pairwise similarity is calculated across 3 containers
  • Aggregated metrics include:
  • Average similarity → overall clustering
  • Minimum similarity → identifies poorest match (key inefficiency signal)

Minimum similarity highlights assignments with at least one poorly matched container.

from itertools import combinations

def jaccard(a,b):
    return len(a & b)/len(a | b) if a | b else 0

results = []

for list_id, group in container_data.groupby('LIST_ID'):

    records = group.to_dict('records')

    sims_sku = []
    sims_loc = []
    sims_aisle = []

    for c1, c2 in combinations(records, 2):
        sims_sku.append(jaccard(c1['SKU_ID'], c2['SKU_ID']))
        sims_loc.append(jaccard(c1['FROM_LOC_ID'], c2['FROM_LOC_ID']))
        sims_aisle.append(jaccard(c1['aisle'], c2['aisle']))

    results.append({
        'LIST_ID': list_id,
        'avg_sku_similarity': sum(sims_sku)/3,
        'avg_location_similarity': sum(sims_loc)/3,
        'avg_aisle_similarity': sum(sims_aisle)/3,
        'min_sku_similarity': min(sims_sku),
        'min_location_similarity': min(sims_loc),
        'min_aisle_similarity': min(sims_aisle)
    })

results_df = pd.DataFrame(results)

Key Findings & Answers¶

1. SKU Similarity

  • Average similarity is low (~12%), indicating weak clustering

2. Consignment Structure

  • No clear improvement from grouping by consignment
  • Consignment does not align with SKU similarity

3. Performance Relationship

  • Higher similarity leads to faster completion
  • Lower similarity results in slower and less predictable performance

4. Travel Efficiency (Proxy)

  • Location similarity correlates similarly to SKU similarity
  • Indicates spatial clustering is critical for efficiency

5. Inefficiency Prevalence

  • ~76% of assignments contain at least one poorly matched container

Limitations¶

This analysis does not include direct measurement of travel distance.

Instead, location similarity is used as a proxy for spatial efficiency.

Limitations include:

  • No exact travel paths
  • No warehouse layout constraints
  • No congestion modelling

Despite this, strong correlation between similarity and duration supports the validity of the proxy.

Conclusion & Recommendations¶

Assignments are currently not optimally structured, with widespread low similarity.

Key implications:

  • Poor clustering drives inefficiency
  • Location matters as much as SKU
  • Consignment grouping is ineffective

Recommendations:

  • Optimise assignments using location proximity
  • Combine SKU and spatial clustering
  • Reduce low-similarity assignments

Further Work:

  • Incorporate travel distance data
  • Analyse pick path optimisation
  • Improve assignment algorithms
